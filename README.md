# Devtools

Проект демонстрирует настройку Gradle, работу с пакетами, record и юнит-тестирование.

---

## 🚀 Quick Start

### Запуск через Gradle Tool Window
1. Открой `View → Tool Windows → Gradle`
2. Разверни дерево:
   - `Tasks → application → run` — запуск приложения
   - `Tasks → verification → test` — запуск тестов
3. Дважды кликни на задачу или нажми иконку Run ▶️

### Запуск через Run Anything
- Нажми **Ctrl + Ctrl** (два раза)
- Введи:
  - `gradle run` — запуск приложения
  - `gradle test` — запуск тестов
  - `gradle build` — полная сборка проекта

### Запуск через терминал
```bash
./gradlew run
./gradlew test
./gradlew build
```

## Пакеты

**`ru.mentee.power`** — корневой пакет проекта.

**Зачем нужен пакет:**
- Предотвращает конфликты имён (классы с одинаковыми названиями из разных модулей не пересекаются)
- Организует код по функциональности
- Правило обратного доменного имени (`ru.mentee.power`) гарантирует глобальную уникальность

**Структура:**
```
src/
└── main/
    └── java/
        └── ru/
            └── mentee/
                └── power/
                    ├── ProgressDemo.java      # точка входа
                    └── MenteeProgress.java    # record с бизнес-логикой
src/
└── test/
    └── java/
        └── ru/
            └── mentee/
                └── power/
                    └── MenteeProgressTest.java   # JUnit-тесты
```

## Таблица полей (record `MenteeProgress`)

| Поле | Тип | Описание |
|------|-----|----------|
| `menteeName` | `String` | Имя студента |
| `sprintNumber` | `int` | Номер текущего спринта |
| `plannedHoursPerWeek` | `int` | Запланированное количество часов на спринт |

### Метод `readyForSprint()`
Возвращает `true`, если `plannedHoursPerWeek >= 3` (порог готовности к спринту).  
Иначе — `false`.

## Git локальный цикл

### Правило веток: feature/DVT‑X
- **master** — чистая ветка для стабильных версий (не коммитим напрямую)
- **feature/DVT‑X** — ветки под конкретные задачи (где X — номер урока)

### Локальный цикл (init → commit → branch)
```bash
# 1. Инициализация репозитория
git init

# 2. Проверка статуса
git status

# 3. Добавление файлов в staging
git add .

# 4. Коммит в master
git commit -m "Initial commit"

# 5. Создание feature-ветки
git checkout -b feature/DVT-3

# 6. Работа в ветке → коммиты
git add .
git commit -m "Добавить фичу"

# 7. Просмотр истории
git log --oneline
```

# 📐 Кодстайл-гайд (Checkstyle)

## 📏 Форматирование

| Правило | Требование |
|---------|------------|
| **Отступы** | 4 пробела (не табуляция!) |
| **Длина строки** | Максимум 100 символов |
| **Перенос строк** | Отступ для перенесённых строк — 8 пробелов |
| **Пробелы** | После `if`, `for`, `while`, `catch` — обязательно |
| **Фигурные скобки** | Открывающая на той же строке, закрывающая на новой |
| **Пустые блоки** | Только как `{}` (без пробелов внутри) |
| **Пробелы в конце строк** | Запрещены |

---

## 📦 Импорты

- **Без звёздочек** (`import java.util.*` — ❌)
- Сначала **статические** импорты, потом **обычные**
- Внутри группы — **по алфавиту**
- Пустая строка **между** группами импортов

---

## 🏷️ Именование

| Элемент | Формат | Пример |
|---------|--------|--------|
| **Классы, интерфейсы, enum** | `UpperCamelCase` | `UserService` |
| **Методы** | `lowerCamelCase` | `getUserById()` |
| **Переменные** | `lowerCamelCase` | `userName` |
| **Константы** | `UPPER_SNAKE_CASE` | `MAX_SIZE` |
| **Пакеты** | только строчные | `ru.mentee.power` |
| **Параметры** | `lowerCamelCase` | `userId` |
| **Типы-параметры (generic)** | одна заглавная буква | `T`, `E`, `K`, `V` |

**Важно:** Сокращения (ID, URL) — только **одна** заглавная буква:  
✅ `getUserId()` ❌ `getUserID()`

---

## 🧩 Структура класса

1. **Пакет** (`package`)
2. **Импорты**
3. **Пустая строка**
4. **Класс / интерфейс / enum**
5. **Поля** → **Конструкторы** → **Методы** (в этом порядке)

---

## 📝 Javadoc

- **Обязателен** для публичных (`public`) классов и методов
- Порядок тегов: `@param` → `@return` → `@throws` → `@deprecated`
- Первая строка — **краткое описание** (с заглавной буквы)

---

## 🚫 Запрещено

| Что | Почему |
|-----|--------|
| `*` в импортах | Засоряет пространство имён |
| Голые `TODO:` | Только `TODO:` (двоеточие обязательно) |
| `switch` без `default` | Всегда должен быть |
| Пустые `catch` без комментария | Должна быть причина |
| `finalize()` | Устарел и опасен |

---

## 🧹 Остальное

- **Один оператор на строку**
- **Одна переменная на строку**
- **Массивы:** `int[] arr` (не `int arr[]`)
- **Модификаторы:** `public` → `protected` → `private` → `static` → `abstract` → `final`
- **Аннотации:** на отдельной строке (кроме `@Test` и однострочных)

---

## 💡 Как пользоваться

1. Настройте IDE под эти правила
2. Используйте автоформатирование (`Ctrl+Alt+L` в IDEA)
3. Перед коммитом — проверьте код через Checkstyle

---

### 🔧 Local Suppression

Если нужно отключить проверку для конкретного места:

```java
// CHECKSTYLE.OFF: ИмяПроверки
// код, который нарушает правило
// CHECKSTYLE.ON: ИмяПроверки
```
